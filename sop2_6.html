1. Inicjalizacja Pamięci (Wzorzec "Otwórz lub Stwórz")

Najczęstszy problem: Jak sprawdzić, czy jestem pierwszym procesem? Ten wzorzec używa flagi O_EXCL. Jeśli plik istnieje, shm_open zwróci błąd, co powie nam, że inicjalizacja już była robiona.
// Struktura dzielona (dostosuj do zadania)
typedef struct {
    pthread_mutex_t mutex;
    int counter;
    int end_flag;
} shared_t;

shared_t* setup_shm(const char* name, int* is_first) {
    // Próba stworzenia (EXCL gwarantuje unikalność pierwszego razu)
    int fd = shm_open(name, O_CREAT | O_EXCL | O_RDWR, 0666);
    *is_first = 1;

    if (fd == -1 && errno == EEXIST) {
        // Już istnieje -> podpinamy się pod istniejącą pamięć
        fd = shm_open(name, O_RDWR, 0666);
        *is_first = 0;
    } else if (fd == -1) {
        perror("shm_open"); exit(1);
    }

    if (*is_first) {
        if (ftruncate(fd, sizeof(shared_t)) == -1) { perror("ftruncate"); exit(1); }
    }

    shared_t* ptr = mmap(NULL, sizeof(shared_t), PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0);
    if (ptr == MAP_FAILED) { perror("mmap"); exit(1); }
    
    close(fd); // Deskryptor niepotrzebny po zmapowaniu
    return ptr;
}

2. Mapowanie pliku log.txt (Bitronka)

Etap 1 Bitronki: Zmapowanie pliku do pamięci zamiast używania fprintf. Pozwala modyfikować plik jak tablicę znaków char[].
char* map_log_file(const char* path, size_t size) {
    int fd = open(path, O_RDWR | O_CREAT | O_TRUNC, 0666);
    if (fd == -1) { perror("open log"); exit(1); }
    
    // Ustawienie fizycznego rozmiaru pliku na dysku
    if (ftruncate(fd, size) == -1) { perror("ftruncate log"); exit(1); }
    
    char* ptr = mmap(NULL, size, PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0);
    if (ptr == MAP_FAILED) { perror("mmap log"); exit(1); }
    
    close(fd);
    return ptr;
}

3. Mutexy: Inicjalizacja i Robustness

W zadaniach SOP mutex musi być PTHREAD_PROCESS_SHARED. Atrybut ROBUST pozwala przejąć mutex, gdy proces padnie (np. przez abort() w Bitronce).
void init_mutex(pthread_mutex_t* m) {
    pthread_mutexattr_t attr;
    pthread_mutexattr_init(&attr);
    
    // 1. Dzielenie między PROCESAMI (nie tylko wątkami)
    pthread_mutexattr_setpshared(&attr, PTHREAD_PROCESS_SHARED);
    
    // 2. Obsługa nagłej śmierci właściciela
    pthread_mutexattr_setrobust(&attr, PTHREAD_MUTEX_ROBUST);
    
    pthread_mutex_init(m, &attr);
    pthread_mutexattr_destroy(&attr);
}

// Bezpieczne blokowanie (Etap 4 Bitronki / Etap 4 Całkowania)
void safe_lock(pthread_mutex_t* m) {
    int rc = pthread_mutex_lock(m);
    if (rc == EOWNERDEAD) {
        // Właściciel zginął, naprawiamy stan mutexa
        pthread_mutex_consistent(m);
        printf("[%d] Mutex recovered after dead owner\n", getpid());
    }
}

4. Obsługa Sygnałów i "Czyste Wyjście"

Kluczowe dla zadania 3 Całkowania. Program nie może zginąć brutalnie, musi dokończyć obliczenia (batch) i dopiero wyjść.
// Zmienna atomowa (bezpieczna dla przerwań)
volatile sig_atomic_t keep_running = 1;

void sig_handler(int sig) {
    if (sig == SIGINT) {
        printf("\n[%d] Received SIGINT, finishing current batch...\n", getpid());
        keep_running = 0;
    }
}

// W main():
struct sigaction sa;
memset(&sa, 0, sizeof(sa));
sa.sa_handler = sig_handler;
sigaction(SIGINT, &sa, NULL);

// W pętli:
while (keep_running) {
    // Obliczaj...
    if (sh->stop_signal) break; // Reakcja na sygnał od Kierownika (Bitronka)
}

5. Cykl Życia: Tworzenie i Sprzątanie

Prawidłowy sposób tworzenia M procesów i czekania na nie. EXIT_SUCCESS u dzieci zapobiega ich "ucieczce" do dalszej części kodu.
void create_workers(int n, shared_t* sh) {
    for (int i = 0; i < n; i++) {
        pid_t pid = fork();
        if (pid == 0) {
            worker_logic(i, sh); // Logika pracownika
            exit(EXIT_SUCCESS);  // KONIECZNIE EXIT
        } else if (pid < 0) {
            perror("fork");
        }
    }
    
    // Rodzic czeka na wszystkie dzieci (Kierownik)
    while (wait(NULL) > 0); 
    printf("All workers finished.\n");
}

6. Precyzyjne Usypianie (ms_sleep)

Zwykły sleep() usypia na całe sekundy. Zadania wymagają np. 100ms.
void ms_sleep(int milis) {
    struct timespec ts;
    ts.tv_sec = milis / 1000;
    ts.tv_nsec = (milis % 1000) * 1000000L;
    nanosleep(&ts, NULL);
}

7. Słownik Pułapek (Dlaczego program nie działa?)

  • Error: Bus Error Zazwyczaj zapomniałeś o ftruncate lub rozmiar mmap jest mniejszy niż Twoja struktura.
  • Error: Deadlock Proces zablokował mutex i zginął, a Ty nie masz ROBUST.
  • Leaking SHM Po każdym teście wyczyść system: rm /dev/shm/*.
  • O_TRUNC Pamiętaj, że O_TRUNC przy otwieraniu pliku do mapowania czyści jego zawartość!

8. Algorytm rozwiązywania zadań (Workflow)

Lista kroków, które musisz przejść w każdym zadaniu typu Bitronka / Całkowanie:

Etap 1: Setup i SHM

  • Definicja Struktury: Stwórz typedef struct zawierającą wszystkie dane (mutexy, liczniki, flagi zakończenia, tablice danych).
  • Semafor bramkowy: Jeśli zadanie to "Całkowanie" (wiele procesów niezależnych), otwórz nazwany semafor, aby tylko pierwszy proces wykonał ftruncate i memset.
  • O_EXCL vs O_RDWR: Użyj shm_open z O_EXCL, aby wykryć, czy jesteś procesem nr 1 (inicjatorem).
  • Mapowanie pliku: Jeśli zadanie wymaga mapowania pliku (Bitronka), otwórz plik przez open, nadaj mu rozmiar przez ftruncate i zmapuj do char*.

Etap 2: Procesy i Logika Dzieci

  • Petla Fork: Stwórz pętlę tworzącą M procesów. Każde dziecko musi zakończyć się przez exit(0), by nie wrócić do pętli rodzica.
  • Inicjalizacja Mutexów: Jeśli jesteś pierwszym procesem, zainicjalizuj mutexy z flagami PTHREAD_PROCESS_SHARED i PTHREAD_MUTEX_ROBUST.
  • Praca Dziecka: Zaimplementuj pętlę pracy (np. 10 prób zamiany produktów). Każda operacja na pamięci dzielonej MUSI być otoczona pthread_mutex_lock i unlock.

Etap 3: Zarządzanie (Kierownik / Sygnały)

  • Proces Kierownika: Stwórz osobny proces (lub niech to będzie Rodzic), który monitoruje stan w pętli while(1) i śpi przez usleep / nanosleep.
  • Sygnały: Zainstaluj handlery sigaction. Handler powinien tylko ustawiać flagę volatile sig_atomic_t keep_running = 0;.
  • Komunikacja: Kierownik kończy pracę dzieci, ustawiając flagę w pamięci dzielonej (np. sh->stop = 1;).

Etap 4: Obsługa Błędów i Śmierci (Robustness)

  • EOWNERDEAD: Zawsze sprawdzaj wynik pthread_mutex_lock. Jeśli zwróci EOWNERDEAD, wywołaj pthread_mutex_consistent i popraw liczniki (np. zdekrementuj liczbę żywych pracowników).
  • Sprzątanie: Ostatni kończący proces MUSI wywołać shm_unlink i sem_unlink. W Bitronce użyj msync przed zakończeniem, aby dane z RAM trafiły do pliku na dysku.
Pamiętaj: W zadaniach z "ostatnim procesem" (Całkowanie), używaj licznika w SHM pod mutexem. Kiedy po sh->counter-- licznik wynosi 0, wykonujesz shm_unlink.
